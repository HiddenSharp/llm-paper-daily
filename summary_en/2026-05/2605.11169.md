## 1. What the Paper Does
- **Background**: Large Language Model (LLM) agents typically operate in a ReAct style in deployed settings, interleaving reasoning, action selection, and observation to solve multi-step decision-making tasks. In scenarios involving repeated related tasks, small action-selection errors can accumulate, leading to wasted tool calls, increased latency, and reduced reliability.
- **Why Existing Work Falls Short**: Current inference-time adaptation methods mainly rely on prompting or retrieval, which indirectly influence behavior through context manipulation. They lack an explicit decision layer to score candidate actions, represent uncertainty, or update online based on action-level feedback, thus failing to provide trackable, fine-grained, and uncertainty-aware adaptation during deployment.

## Core Contributions
- **Proposed OLIVIA**: An inference-time action adaptation framework for ReAct-style agents that models the LLM's final action-selection layer as a contextual linear bandit over candidate actions.
- **Challenge 1: How to achieve direct and efficient action policy adaptation while preserving the underlying reasoning process?**
  - **Difficulty**: Traditional fine-tuning is costly and compromises generality, while prompt adjustments are indirect and hard to optimize quantitatively.
  - **Solution**: OLIVIA uses frozen hidden states as decision contexts and introduces a lightweight linear bandit layer directly at the action-selection interface. This allows for online updates targeted solely at action selection, preserving original reasoning capabilities with low overhead.
- **Challenge 2: How to effectively balance exploration and exploitation to improve sample efficiency without abundant labeled data?**
  - **Difficulty**: Feedback in deployment environments is sparse and noisy; blind exploitation may lead to local optima, while excessive exploration increases error costs.
  - **Solution**: By incorporating Upper-Confidence-Bound (UCB) exploration and explicit uncertainty estimates, OLIVIA enables the agent to efficiently improve its policy based on action-level feedback, minimizing computational overhead while enhancing decision quality.

## Implementation and Evaluation
- **Implementation & Deployment**: OLIVIA was instantiated and evaluated on four benchmarks. The framework requires no LLM parameter updates, maintaining only a lightweight linear model for action scoring and selection, making it suitable for streaming deployment.
- **Evaluation Results**: Experiments demonstrate that OLIVIA consistently outperforms static ReAct baselines and prompt-based inference-time adaptation baselines across all benchmarks.
- **Comparison Advantages**: Compared to prompt or retrieval-based methods that rely on context manipulation, OLIVIA provides an explicit online decision layer. It more effectively utilizes action-level feedback, significantly reducing invalid trajectories caused by early errors and improving task completion rates and efficiency.

## 4. Conclusion
OLIVIA demonstrates that introducing an explicit online decision layer in LLM agents is an effective alternative to purely prompt- or retrieval-based adaptation. By modeling action selection as a contextual linear bandit problem, OLIVIA achieves fine-grained, trackable, and uncertainty-aware inference-time adaptation, offering an efficient, low-overhead path to performance improvement for deployed LLM agents.
