## 1. What the Paper Did
- **Background**: Automation platforms like GitHub Actions and n8n are increasingly integrating Large Language Model (LLM) agents for tasks such as code review and data synchronization, creating "agentic workflows." While beneficial, this introduces new security risks where adversaries can manipulate LLM agents via crafted inputs (e.g., GitHub issue comments) to perform unwanted actions like credential exfiltration or arbitrary command execution.
- **Limitations of Existing Work**: Existing workflow analysis tools rely on static, path-insensitive analysis, failing to account for feasible agent-invocation paths or runtime behavior. Conversely, prior LLM jailbreaking research assumes full control over inputs, which does not hold in agentic workflows where input control is partial and constrained by the agent's runtime capabilities and restrictions.

## Core Contributions
- **Proposed JAW Framework**: The first detection and exploitation framework for hijacking agentic workflows on automation platforms, utilizing a novel approach called "Context-Grounded Evolution."
- **Challenge 1: Identifying feasible invocation paths and input constraints.**
  - **Difficulty**: Complex workflow logic makes it hard for static analysis to determine which paths are reachable and what input conditions trigger specific agent behaviors.
  - **Solution**: JAW employs static path-feasibility analysis to identify feasible agent-invocation paths and the specific input constraints required to trigger them.
- **Challenge 2: Generating effective payloads under limited input control.**
  - **Difficulty**: Attackers cannot fully control the prompt, and agent actions are limited by runtime environments, rendering traditional jailbreak methods ineffective.
  - **Solution**: JAW uses dynamic prompt-provenance analysis to track how inputs are embedded into the LLM context and capability analysis to identify runtime restrictions. It then iteratively synthesizes and evolves payloads grounded in these contexts for end-to-end exploitation.

## Implementation and Evaluation
- **Evaluation Results**: JAW was evaluated on GitHub workflows and n8n templates. It successfully hijacked 4,174 GitHub workflows and 8 n8n templates, demonstrating capabilities such as leaking user credentials.
- **Scope**: Findings covered 15 widely-used GitHub Actions, including official actions for Claude Code, Gemini CLI, Qwen CLI, and Cursor CLI, as well as two official n8n nodes.
- **Comparison & Impact**: As the first study in this domain, JAW addresses a previously unexplored security gap. All findings were responsibly disclosed, leading to acknowledgments, fixes, and bug bounties from major vendors including GitHub, Google, and Anthropic.

#### Summary
This paper systematically investigates the security risks of agentic workflows in automation platforms, highlighting the inadequacy of existing static analysis and traditional LLM jailbreaking methods. By proposing the JAW framework, which combines static path analysis, dynamic prompt tracking, and runtime capability analysis, the authors effectively detect and exploit vulnerabilities in agentic workflows. Large-scale experiments confirm the prevalence and severity of these risks, prompting significant security improvements from industry leaders.
