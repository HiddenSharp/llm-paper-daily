## 1. What the Paper Does
- **Background**: Table processing (including cleaning, transformation, augmentation, and matching) is a foundational yet error-prone stage in real-world data pipelines. While LLM-based approaches show promise, they often struggle due to ambiguous instructions, complex task structures, and a lack of structured feedback, resulting in syntactically correct but semantically flawed code.
- **Why Existing Work Falls Short**: Current methods are often narrowly scoped to isolated operations (e.g., imputation or error correction) and cannot handle composite workflows. Broader frameworks prioritize other tasks like AutoML or table QA over comprehensive table processing. Furthermore, existing systems lack dynamic semantic understanding and closed-loop feedback mechanisms, making it difficult to translate ambiguous user intents into robust, governance-compliant code.

## Core Contributions
- **Proposed ProfiliTable**: An autonomous multi-agent framework centered on dynamic profiling, which constructs and iteratively refines a unified execution context through interactive exploration, knowledge-augmented synthesis, and feedback-driven refinement.
- **Challenge 1: Ambiguous Instructions and Lack of Semantic Understanding**. The difficulty lies in vague natural language instructions and complex domain semantics in tabular data, which hinder direct accurate code generation by LLMs. ProfiliTable addresses this by integrating a **Profiler** agent that performs ReAct-style data exploration to build deep semantic understanding and clarify the execution context.
- **Challenge 2: Code Generation Errors and Lack of Self-Correction**. One-shot code generation often contains logical errors or fails to adhere to operator specifications, lacking effective debugging signals. ProfiliTable solves this by using a **Generator** to synthesize task-aware code from curated operators, combined with an **Evaluator-Summarizer** loop that injects execution scores and diagnostic insights for closed-loop refinement, ensuring semantic correctness and robustness.

## Implementation and Evaluation
- **Implementation**: ProfiliTable integrates three core components: (i) a Profiler for data exploration; (ii) a Generator for code synthesis using retrieved operators; and (iii) an Evaluator-Summarizer loop for feedback. It aims to automate diverse table tasks from cleaning to matching.
- **Evaluation Results**: Extensive experiments were conducted on a diverse benchmark covering 18 tabular task types. Results demonstrate that ProfiliTable consistently outperforms strong baselines.
- **Comparison**: ProfiliTable shows significant superiority, particularly in complex multi-step scenarios, highlighting the critical role of dynamic profiling in reliably translating ambiguous user intents into robust table transformations compared to existing methods.

## 4. Conclusion
ProfiliTable effectively addresses the limitations of existing LLM-based table processing methods when facing ambiguous instructions and complex tasks by introducing dynamic profiling and multi-agent collaboration. Its core innovation lies in constructing an "explore-synthesize-refine" closed-loop workflow, significantly improving the automation level and semantic accuracy of code generation for table data processing. It provides a robust and compliant solution for complex data cleaning and transformation tasks in real-world scenarios.
