## 1. What the Paper Did
- **Background**: As Large Language Models (LLMs) and AI agents rapidly advance, their potential to reshape cybersecurity necessitates rigorous evaluation. A critical capability is "exploitation": transforming a known vulnerability into a concrete security impact, such as unauthorized file access or code execution.
- **Why Existing Work Falls Short**: Prior benchmarks focus on vulnerability reproduction, patch generation, or CTF solving, which primarily require source-level reasoning. However, real-world exploitation demands low-level runtime reasoning (e.g., memory layout, register states), long-horizon planning, and the ability to bypass modern mitigations (e.g., ASLR, stack canaries). The lack of systematic evaluation for this high-order capability leaves a gap in understanding the true offensive boundaries of AI agents.

## Core Contributions
- **Proposed ExploitGym**: A large-scale, diverse, and realistic benchmark for evaluating AI agent exploitation capabilities, comprising 898 instances from real-world vulnerabilities across user-space programs, Google's V8 JavaScript engine, and the Linux kernel.
- **Challenge 1: Complexity of Low-Level Program Reasoning and Runtime Adaptation**. The difficulty lies in requiring agents to understand heap metadata, stack frames, and virtual memory mappings while dynamically adapting strategies at runtime. ExploitGym addresses this by providing reproducible containerized environments where agents must progressively extend an initial vulnerability-triggering input into a full exploit, forcing fine-grained low-level reasoning.
- **Challenge 2: Long-Horizon Planning and Bypassing Modern Defenses**. The challenge involves chaining multiple primitives and bypassing mitigations like ASLR and sandboxes, demanding high logical coherence and long-term memory. The benchmark isolates the impact of these defenses by varying security configurations across instances, quantifying agent performance under adversarial conditions.

## Implementation and Evaluation
- **Implementation**: ExploitGym includes 898 instances across three domains, packaged in reproducible containerized environments with variable security protections. Experiments were conducted under trusted-access programs with safeguards disabled to measure capability boundaries.
- **Evaluation Results**: While exploitation remains challenging, frontier models successfully exploited a non-trivial fraction of vulnerabilities. The strongest configurations, Anthropic's Claude Mythos Preview and OpenAI's GPT-5.5, produced working exploits for 157 and 120 instances, respectively.
- **Comparison & Insights**: Unlike prior source-level benchmarks, ExploitGym reveals that models retain significant success rates even when widely used defenses are enabled. This indicates that current defenses are not absolute barriers against AI-driven exploitation and highlights the growing sophistication of agent-based attacks.

## 4. Conclusion
ExploitGym fills a critical gap in AI security evaluation by focusing on vulnerability exploitation. The study demonstrates that frontier AI models possess significant capability to convert vulnerabilities into real-world attacks, even against modern mitigations. This establishes ExploitGym as an effective testbed for defensive workflows while underscoring the urgent cybersecurity risks posed by increasingly capable AI agents, calling for enhanced regulation and defense research.
