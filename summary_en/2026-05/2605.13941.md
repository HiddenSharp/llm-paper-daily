## 1. What the Paper Does
- **Background**: Long-term memory is critical for LLM agents operating across multiple sessions. As memory stores grow in scale and complexity, different query types (e.g., factual lookup, temporal reasoning) require distinct retrieval strategies.
- **Why Existing Work Fails**: Current memory systems (e.g., MemGPT, Mem0) focus on content organization or maintenance but assume a fixed retrieval infrastructure. Scoring functions, fusion strategies, and generation policies remain frozen after deployment. This static configuration creates a mismatch with evolving memory stores and diverse query needs, leading to suboptimal performance over time.

## Core Contributions
- **Proposed EvolveMem**: A self-evolving memory architecture that utilizes AutoResearch to co-evolve both stored knowledge and the retrieval mechanism. It exposes the full retrieval configuration as a structured action space optimized by an LLM-powered diagnosis module.
- **Challenge 1: How to automatically diagnose root causes of retrieval failures and generate effective configuration adjustments?**
  - **Difficulty**: Retrieval failures stem from various factors (e.g., keyword mismatch, context limits), and the configuration space is vast, making manual tuning inefficient and incomplete.
  - **Solution**: An LLM-driven diagnosis module reads per-question failure logs in each evolution round, identifies root causes, and proposes targeted configuration changes, automating the research cycle previously done manually.
- **Challenge 2: How to ensure stability during self-evolution and prevent performance degradation from harmful configurations?**
  - **Difficulty**: Automatic exploration can lead to volatile performance or catastrophic regression without safety mechanisms.
  - **Solution**: A guarded meta-analyzer applies "revert-on-regression" and "explore-on-stagnation" safeguards, ensuring only performance-enhancing configurations are retained, thus balancing stability and optimization.

## Implementation and Evaluation
- **Implementation**: EvolveMem starts from a minimal baseline and converges autonomously through a closed-loop process (Evaluate-Diagnose-Propose-Guard). Code is available on GitHub.
- **Evaluation Results**:
  - On **LoCoMo**, EvolveMem outperforms the strongest baseline by **25.7%** relative and achieves a **78.0%** relative improvement over the minimal baseline.
  - On **MemBench**, it exceeds the strongest baseline by **18.9%** relative.
- **Comparison & Transferability**: Evolved configurations show positive transfer across benchmarks, indicating the capture of universal retrieval principles rather than benchmark-specific heuristics, surpassing traditional fixed-retrieval systems.

## 4. Conclusion
EvolveMem addresses the limitation of fixed retrieval mechanisms in traditional memory systems by introducing a self-evolving architecture. Through an LLM-driven AutoResearch process, it autonomously diagnoses failures, adjusts configurations, and safeguards evolution. Experiments demonstrate significant improvements over state-of-the-art baselines and strong cross-task generalization, offering a new paradigm for truly adaptive LLM agent memory.
