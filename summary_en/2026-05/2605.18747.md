## 1. What the Paper Does
- **Background**: With Large Language Models (LLMs) demonstrating superior capabilities in code understanding and generation, the role of code in agentic systems is evolving. Code is no longer just a final output artifact but increasingly serves as the operational substrate for agent reasoning, action, environment modeling, and execution-based verification.
- **Limitations of Existing Work**: Previous studies often treat code merely as a target output or simple tool scripts, lacking a unified perspective to systematically understand how code supports the entire agent lifecycle. This fragmented view hinders the development of executable, verifiable, and stateful complex agent systems.

## Core Contributions
- **Proposed a Unified Framework**: The paper introduces "Code as Agent Harness," a unified view centering code as the foundation of agent infrastructure, structured around three interconnected layers.
- **Challenge 1: How to translate internal reasoning into verifiable external steps?**
  - **Difficulty**: LLM internal reasoning is often a black box, prone to hallucinations and error accumulation, making direct verification difficult.
  - **Solution**: Through the "Harness Interface" layer, code externalizes internal reasoning into executable intermediate steps. This allows each logical step to be compiled, run, and verified, enabling precise tracking and error localization.
- **Challenge 2: How to maintain state consistency and reliability in long-horizon tasks?**
  - **Difficulty**: Agents tend to lose context over long interactions and lack adaptive capabilities in dynamic environments.
  - **Solution**: Through the "Harness Mechanisms" layer, the paper integrates planning, memory, and tool use. Code manages short-term working memory and long-term experience reuse, while feedback-driven control loops (e.g., iterative debugging, runtime adjustments) optimize execution paths for reliability.

## Implementation and Evaluation
- **Survey Nature**: As a systematic survey, this paper does not propose a single new algorithm with traditional experimental benchmarks. Its "evaluation" lies in the taxonomic summary and gap analysis of existing representative methods.
- **Scope and Comparison**: The paper covers diverse scenarios from single-agent to multi-agent systems, including coding assistants, GUI/OS automation, embodied agents, and scientific discovery. Unlike previous surveys focusing solely on code generation accuracy, this work compares methods based on "Harness Engineering" aspects, such as state management, execution verification mechanisms, and the use of shared code artifacts in multi-agent collaboration.
- **Open Challenges**: The paper highlights current limitations, such as the over-reliance on final task success metrics, verification difficulties under incomplete feedback, and consistency issues in shared states across multiple agents, providing a clear roadmap for future research.

## 4. Conclusion
By proposing the "Code as Agent Harness" perspective, this paper systematically reviews the core role of code as a basis for reasoning, an interface for action, and a model for the environment in agent systems. It not only summarizes current technical progress but also outlines the path from single-agent to multi-agent expansion, identifying key challenges in executability, verifiability, and statefulness, thereby providing a theoretical framework and research direction for building more reliable and transparent AI agent systems.
