## 1. What the Paper Does
- **Background**: Relational learning aims to automate predictive modeling over relational databases. Traditionally, this requires data analysts to manually write complex SQL queries for feature construction (joins, aggregations), a process that is time-consuming and demands significant domain expertise.
- **Why Existing Work Falls Short**:
    - **Tabular Methods** (e.g., Tabular Foundation Models): Require flattening relational data, often relying on generic aggregations or extensive manual feature engineering, failing to capture task-specific relational interactions.
    - **Graph-based Models** (e.g., GNNs): Rely on fixed mechanisms for extracting subgraphs, where the context scope (e.g., neighborhood size) is pre-defined and may not align with downstream tasks.
    - **Sequence-based Approaches** (e.g., direct LLM application): Offer flexible reasoning but lack inductive biases naturally aligned with structured relational prediction, often suffering from high latency, cost, and lack of interpretability.

## Core Contributions
- **Proposed RelAgent**: An LLM-based autonomous data scientist agent for relational learning. It frames relational learning as an agentic search process over query-based features and predictive models, ultimately producing a predictor composed of SQL queries and a classical model.
- **Challenge 1: How to automatically discover high-quality, task-relevant relational features?**
    - **Difficulty**: The feature space is vast, involving complex combinations of joins, filters, and aggregations. Blind search is inefficient and error-prone.
    - **Solution**: RelAgent employs an LLM agent in the search phase that iteratively proposes and refines SQL feature programs. It utilizes three tools: database exploration via read-only SQL, program validation by executing candidates on a validation set to get performance metrics, and inspection of past trials in an Evaluation Workspace. This feedback loop allows the agent to optimize feature queries and model configurations effectively.
- **Challenge 2: How to achieve efficient, interpretable, and scalable deployment?**
    - **Difficulty**: Many LLM-based methods require model calls during inference, leading to high latency, cost, and stochastic results. Black-box models lack intrinsic interpretability.
    - **Solution**: RelAgent adopts a two-phase design. During inference, no LLM calls are made; instead, the SQL queries and classical model generated in the search phase are executed directly. This ensures fast, deterministic, and intrinsically interpretable predictions (features are human-readable SQL queries), enabling scalable deployment using standard database systems.

## Implementation and Evaluation
- **Implementation & Deployment**: The system separates the search phase (LLM-driven tool interaction to generate SQL feature programs and select models) from the inference phase (execution of generated SQL and classical models without LLM). This allows the final predictor to be seamlessly integrated into existing standard database infrastructures.
- **Evaluation & Comparison**:
    - RelAgent demonstrates competitive or superior performance compared to traditional Graph Neural Networks (GNNs) and tabular foundation models across various relational learning benchmarks.
    - Compared to methods that directly apply LLMs to serialized data, RelAgent offers higher determinism and interpretability.
    - A key advantage is that the generated features are explicit SQL queries, which are not only performant but also auditable and understandable by humans, addressing trust issues associated with black-box models.
    - Since the inference phase excludes LLM computation, RelAgent achieves significantly faster inference speeds and lower costs compared to methods requiring LLM calls at runtime.

## 4. Conclusion
RelAgent innovatively uses LLMs as search agents rather than direct predictors for automating feature engineering in relational learning. By decoupling search from execution, it leverages the strong code generation and reasoning capabilities of LLMs to explore complex feature spaces while retaining the efficiency, determinism, and interpretability of traditional SQL and classical models. This approach effectively bridges the gap between flexibility and interpretability found in existing graph and tabular methods, offering a scalable and practical pathway for automated machine learning over relational databases.
