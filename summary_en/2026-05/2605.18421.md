## 1. What the Paper Did
- **Background**: While Large Language Model (LLM) agents have advanced in reasoning, planning, and execution, memory capability (storing, updating, and retrieving information over time) remains essential but under-evaluated. Existing benchmarks primarily focus on reasoning and execution, lacking a systematic assessment of memory mechanisms.
- **Why Existing Work Falls Short**: Current benchmarks do not provide a unified framework for evaluating memory, often failing to distinguish between different memory dimensions (e.g., episodic scope and content type). Furthermore, simple long-context baseline methods are highly effective, masking the specific benefits of specialized memory systems in self-evolving scenarios, making it difficult to gauge the true efficacy of different memory approaches.

## Core Contributions
- **Proposed EvoMemBench**: A unified benchmark for evaluating agent memory from a self-evolving perspective, organized along two axes: memory scope (in-episode vs. cross-episode) and memory content (knowledge-oriented vs. execution-oriented).
- **Challenge 1: How to systematically decouple and evaluate different dimensions of memory?**
  - **Difficulty**: Memory involves short-term state retention, long-term knowledge accumulation, and the distillation of execution experience, making it hard to quantify the contribution of specific modules when mixed.
  - **Solution**: The paper designs a two-dimensional evaluation matrix, categorizing tasks into in-episode/cross-episode and knowledge/execution-oriented, thereby isolating variables to precisely assess the effectiveness of different memory mechanisms in specific scenarios.
- **Challenge 2: How to fairly compare diverse memory methods with strong long-context baselines under a standardized protocol?**
  - **Difficulty**: Memory methods vary significantly in implementation, and powerful long-context models can obscure the value of dedicated memory modules, lacking a uniform standard for comparison.
  - **Solution**: Established a standardized evaluation protocol comparing 15 representative memory methods against strong long-context baselines, clarifying the specific performance boundaries where memory systems provide gains, particularly when context is insufficient or tasks are complex.

## Implementation and Evaluation
- **Implementation & Deployment**: EvoMemBench code is open-sourced. The study evaluated 15 mainstream memory methods under a unified protocol, comparing them with robust long-context baselines.
- **Evaluation Results**:
  1. Current memory systems are not yet a general solution; long-context baselines remain highly competitive in many scenarios.
  2. Memory mechanisms provide the most significant help when the current context is insufficient or tasks are difficult.
  3. No single memory form works consistently across all settings.
  4. **Comparative Analysis**: Retrieval-based methods remain strong in knowledge-intensive settings, whereas procedural and long-term memory methods are more effective for execution-oriented tasks, provided the stored experience matches the task structure.

## 4. Conclusion
This paper introduces EvoMemBench, filling a gap in the evaluation of LLM agent memory. By systematically assessing memory from a self-evolving perspective (in-episode/cross-episode, knowledge/execution), it reveals the limitations and applicable scenarios of current memory systems. The study shows that while long-context models are powerful, specialized memory mechanisms offer irreplaceable value in specific complex tasks and cross-task experience reuse, providing direction for future development of more efficient agent memory systems.
