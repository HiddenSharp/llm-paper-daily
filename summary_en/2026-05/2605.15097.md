## 1. What the Paper Does
- **Background**: Detecting memory corruption vulnerabilities (MCVs) in stripped binaries is challenging as it requires recovering object semantics, interprocedural propagation, and feasible trigger conditions from low-level, semantically lossy representations.
- **Limitations of Existing Work**: Recent LLM-based approaches improve code understanding but lack reliable detection capabilities in stripped binaries because they do not explicitly ground findings in memory-relevant program semantics or provide runtime feasibility evidence beyond lossy lifted or decompiled representations.

## Core Contributions
- **Proposed Veritas**, a semantically grounded agentic framework for binary memory corruption vulnerability detection, combining a static slicer, a dual-view LLM detector, and a multi-agent validator.
- **Challenge 1: Accurately recovering interprocedural value flows and object semantics from low-level representations.**
  - **Difficulty**: Binaries lack high-level semantic information; traditional static analysis struggles with precise pointer tracking, while pure LLM methods may hallucinate or miss low-level details.
  - **Solution**: Veritas employs a static slicer (defuse) over RetDec-lifted LLVM IR to reconstruct interprocedural value-flow relations (def-use, calls, returns, globals, pointers) and emits compact, witness-backed flow objects, providing a solid semantic foundation for downstream analysis.
- **Challenge 2: Ensuring detected vulnerability candidates are runtime-feasible and minimizing false positives.**
  - **Difficulty**: Static analysis or LLM reasoning may identify theoretically existing but practically unreachable or untriggerable paths, leading to high false positive rates.
  - **Solution**: A multi-agent validator grounds detector hypotheses in debugger-visible artifacts and concrete runtime evidence through guided debugging, breakpoint inspection, and memory-checking oracles to confirm or reject candidates.

## Implementation and Evaluation
- **Implementation**: Veritas is implemented as a modular pipeline featuring a static slicer, a dual-view LLM detector (reasoning step-by-step over decompiled C and selective LLVM IR), and a multi-agent validator.
- **Evaluation Results**:
  - Achieved **90% recall** on a curated benchmark of real-world binary vulnerability cases.
  - **False Positive Assessment**: Exhaustive validation of 623 candidates resulted in **zero false positives** in the tractable subset; an additional audit of larger cases identified only two manually confirmed false positives.
  - **Real-world Impact**: Discovered a previously unknown Apple vulnerability (0day), which was confirmed and assigned a CVE.
- **Comparison**: Compared to existing LLM baselines, Veritas significantly improves reliability and accuracy by integrating semantic grounding and runtime validation, addressing the high false positive rates and poor feasibility judgment of purely static or LLM-based methods.

## 4. Conclusion
Veritas demonstrates the effectiveness of "semantic grounding" as an operational design principle for binary vulnerability detection. By combining the precision of static analysis, the reasoning capabilities of LLMs, and the reliability of dynamic validation, Veritas efficiently and accurately detects memory corruption vulnerabilities in stripped binaries, proving its potential for practical security auditing through the discovery of critical unknown vulnerabilities.
