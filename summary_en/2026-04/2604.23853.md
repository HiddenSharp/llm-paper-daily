## 1. What the Paper Did
- **Background**: LLM Agents can be improved by distilling reusable "skills" (structured instruction packages) from execution trajectories without modifying model weights. However, existing skill-distillation pipelines lack a key signal: the specific cost of each step.
- **Why Existing Work Falls Short**: Current methods typically split trajectories into successes and failures to extract rules, failing to distinguish between "adding missing steps to fix bugs" and "removing expensive but irrelevant steps to cut costs." While observability tools like LangSmith and Langfuse track token counts and costs, they provide dashboards for human operators rather than compact, machine-consumable intermediate representations (e.g., structured data with per-step costs and redundancy flags) needed for automated distillation.

## Core Contributions
- **Proposed ClawTrace Platform and CostCraft Pipeline**: ClawTrace is an open-source tracing platform that compiles agent sessions into compact YAML TraceCards; CostCraft is a distillation pipeline built on this to generate three types of skill patches: Preserve, Prune, and Repair.
- **Challenge 1: Lack of fine-grained cost attribution prevents efficiency optimization**. It is difficult to identify which high-cost steps contribute nothing to the outcome. The paper addresses this by using ClawTrace to record specific USD costs and token counts for every LLM call, tool use, and sub-agent spawn, marking redundancies in TraceCards to allow precise identification of waste.
- **Challenge 2: Asymmetry in cross-task generalization of distilled skills**. "Preserve rules" trained on specific benchmark conventions may cause performance regressions on new tasks. The paper introduces "Prune patches" that use counterfactual arguments to remove invalid high-cost steps, finding that prune rules generalize better across benchmarks, while preserve rules require caution.

## Implementation and Evaluation
- **Implementation**: ClawTrace instruments agent sessions via eight event hooks, linking child agent sessions to parent spans and accounting for cached tokens at actual billed rates. It compiles ~1.5 kB TraceCards containing per-step USD costs, token counts by type, redundant tool-call clusters, and failure/repair markers. The API accepts plain JSON with no framework dependencies.
- **Evaluation Results**:
    - Ablation studies on 30 held-out SpreadsheetBench tasks show that both cost attribution and prune patches independently reduce quality regressions.
    - Testing on 30 unrelated SkillsBench tasks revealed an asymmetry: Prune rules transferred across benchmarks and reduced median costs by 32%, whereas preserve rules caused regressions on new task types due to overfitting to benchmark-specific conventions.
- **Comparison**: Unlike existing tools (e.g., LangSmith) that offer human-readable dashboards, ClawTrace provides a machine-readable compact intermediate representation specifically designed for automated skill distillation.

## 4. Conclusion
This paper addresses the neglect of cost signals in current LLM Agent skill distillation by proposing the ClawTrace tracing platform and TraceCard data format, solving the problem of fine-grained cost attribution. The resulting CostCraft pipeline generates Preserve, Prune, and Repair skill patches. The study highlights the efficiency and generalizability of prune rules in cross-task cost optimization (reducing median costs by 32%), while noting that preserve rules may suffer from overfitting. ClawTrace is released as open infrastructure to advance cost-aware Agent research.
