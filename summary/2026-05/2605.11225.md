## 文章做了什么
- **背景**：基于大语言模型（LLM）的自主代理在规划多步行动轨迹时，经常生成看似连贯但在执行中失败的计划。这种“规划-执行错位”主要由不可行动作、约束违反以及长视野下的误差累积导致。
- **已有工作为什么解决不了**：现有的改进方法缺乏轨迹级别的优化视角。诊断方法仅在事后识别失败而不优化底层计划；迭代细化方法仅修改局部推理步骤或提示词；约束基方法应用局部修复；进化方法虽搜索候选轨迹，但未直接优化由执行结果 induced 的损失。目前缺少一种将完整轨迹视为可优化对象并将执行反馈转化为更新信号的机制。

## 文章的核心贡献点
- **提出了一个 PIVOT (Plan-Inspect-eVOlve Trajectories) 框架**：这是一个自监督框架，通过环境交互迭代地细化轨迹，将轨迹视为可优化对象。
- **挑战1：规划与执行的系统性偏差及误差累积**。难点在于LLM生成的计划在静态视角下有效，但在动态环境中因状态假设错误或动作不可行而失效，且错误会级联放大。PIVOT 通过 INSPECT 阶段执行轨迹并计算结构化损失（包含编码规划-执行差异的文本梯度），随后在 EVOLVE 阶段利用这些信号生成改进的轨迹，从而直接针对执行结果进行优化。
- **挑战2：如何在保证解质量不下降的同时提高计算效率**。难点在于传统的迭代 refinement 方法往往 token 消耗巨大且可能陷入局部最优或质量波动。PIVOT 引入单调接受过程（monotonic acceptance process）确保解质量非递减，并通过高效的轨迹级目标函数（结合目标达成损失、规划-执行发散和执行成本），相比竞争方法减少了3-5倍的 token 消耗。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **实现与部署**：PIVOT 包含四个阶段：PLAN（生成候选轨迹）、INSPECT（执行并计算带文本梯度的结构化损失）、EVOLVE（应用信号产生改进轨迹）、VERIFY（最终全局约束检查）。该框架支持完全自主模式和人机协同（HITL）模式。
- **评估结果**：在 DeepPlanning 和 GAIA 基准测试中达到了最先进（SOTA）性能。
  - **人机协同（HITL）**：建立了强大的上限，约束满足率相对提升高达 94%。
  - **完全自主**：保留了显著的性能增益，证明核心轨迹细化机制在无外部监督下依然有效。
- **对比相关工作**：相比其他细化方法，PIVOT 在计算效率上具有显著优势，所需的 token 数量减少多达 3 到 5 倍，同时解决了现有方法未能处理的轨迹级优化问题。

#### 总结
PIVOT 证明了基于反馈（自监督或人工监督）的轨迹优化是缓解自主代理系统中规划-执行差距的原则性方法。通过将完整轨迹作为离散语言空间中的可优化对象，并结合单调接受机制，PIVOT 不仅在约束满足和任务成功率上取得了显著提升，还大幅降低了计算成本，为构建更可靠的 LLM 代理系统提供了新范式。
