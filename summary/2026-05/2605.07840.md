## 文章做了什么
- **背景**：关系学习（Relational Learning）旨在自动化从关系数据库中进行预测建模的过程。传统上，这需要数据分析师编写复杂的 SQL 查询来构建特征（如连接表、聚合信号等），这一过程耗时且需要深厚的领域专业知识。
- **已有工作为什么解决不了**：
    - **表格方法**（如 Tabular Foundation Models）：需要将关系数据扁平化，往往依赖通用聚合或大量手工特征工程，难以捕捉特定任务的关系交互。
    - **图基于模型**（如 GNNs）：依赖预定义的子图提取机制，其上下文范围（如邻居大小）是固定的，可能无法对齐下游任务需求。
    - **序列方法**（如直接应用 LLM）：虽然推理灵活，但其归纳偏置并非天然适配结构化关系预测，且通常缺乏可解释性和高效部署能力。

## 文章的核心贡献点
- **提出了一个 RelAgent**：一种基于 LLM 的自主数据科学家代理，用于关系学习。它将关系学习转化为对基于查询的特征和预测模型的代理搜索过程，最终生成由 SQL 查询和经典模型组成的预测器。
- **挑战1：如何自动发现高质量且任务相关的关系特征？**
    - **难点**：特征空间巨大，涉及复杂的 join、filter 和 aggregation 组合，盲目搜索效率低且容易出错。
    - **解决方法**：RelAgent 在搜索阶段利用 LLM 代理结合三个工具（数据库探索、验证执行、评估工作区检查）进行迭代式搜索。代理通过执行只读 SQL 探索数据库结构，运行候选程序获取验证集性能反馈，并回顾历史试验结果，从而逐步优化 SQL 特征程序和模型配置。
- **挑战2：如何实现高效、可解释且可扩展的部署？**
    - **难点**：许多 LLM 基方法在推理时仍需调用模型，导致延迟高、成本高且结果具有随机性；黑盒模型缺乏内在可解释性。
    - **解决方法**：RelAgent 采用两阶段设计。在推理阶段，完全不再调用 LLM，而是直接执行搜索阶段生成的 SQL 查询和经典机器学习模型。这使得预测速度快、确定性高，且特征即为人类可读的 SQL 查询，实现了内在可解释性，并可利用标准数据库系统进行规模化部署。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **实现与部署**：系统分为搜索阶段和推理阶段。搜索阶段由 LLM 驱动，使用工具交互生成 SQL 特征程序和选择模型；推理阶段仅执行生成的 SQL 和经典模型，无需 LLM 参与。这种设计使得最终预测器可以无缝集成到现有的标准数据库系统中。
- **Evaluation 结果与对比**：
    - 相比于传统的图神经网络（GNNs）和表格基础模型，RelAgent 在多个关系学习基准测试中展现了竞争力或更优的性能。
    - 与直接使用 LLM 进行序列化处理的方法相比，RelAgent 生成的解决方案具有更高的确定性和可解释性。
    - 关键优势在于其生成的特征是显式的 SQL 查询，不仅性能良好，而且便于人工审查和理解，解决了黑盒模型在工业界部署的信任难题。
    - 由于推理阶段不涉及 LLM 计算，其推理速度显著快于需要在推理时调用 LLM 的方法，且成本更低。

#### 总结
RelAgent 创新性地将 LLM 作为搜索代理而非直接的预测模型，用于自动化关系学习中的特征工程。通过“搜索-执行”分离的架构，它既利用了 LLM 强大的代码生成和推理能力来探索复杂的特征空间，又保留了传统 SQL 和经典模型的高效性、确定性和可解释性。这种方法有效克服了现有图模型和表格方法在灵活性与可解释性之间的权衡，为关系数据库上的自动化机器学习提供了一条可扩展且实用的新路径。
