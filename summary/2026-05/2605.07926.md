## 文章做了什么
- **背景**：随着基于大语言模型（LLM）的代理越来越多地依赖外部工具，评估其在熟悉工作流和短程交互之外的工具 grounding 推理能力变得至关重要。
- **已有工作的不足**：现有的代理基准测试（如 BFCL、Tau2-Bench、SWE-bench 等）主要关注单个 API 调用或特定领域（如旅行规划、客户服务、代码编辑）的多步任务。这些任务通常基于熟悉的领域和重复出现的解决方案模板，导致高性能可能反映的是对领域惯例的记忆或短程反应式工具使用，而非在陌生环境中推断和执行新颖工具使用程序的能力。

## 文章的核心贡献点
- **提出了一个 AgentEscapeBench**：这是一个类似“密室逃脱”风格的基准测试，旨在测试代理在显式长程依赖约束下，推断、执行和修正新颖工具使用程序的能力。
- **挑战1：长程状态跟踪与依赖管理**。难点在于任务定义了工具和物品之间的有向无环图（DAG），要求代理在多个步骤中跟踪逐渐揭示的隐藏状态并传播中间结果。论文通过设计具有不同难度层级（依赖深度）的任务，强制代理必须正确维护上下文状态才能解开最终谜题，从而量化评估这一能力。
- **挑战2：域外（Out-of-Domain）工具推理**。难点在于代理面对的是不熟悉的工具和叙事线索，无法依赖预训练中的常见模式。论文通过构建包含真实外部函数调用的环境，要求代理根据叙事线索推断参数并执行从未见过的工具组合，以此检验其真正的泛化推理能力。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **实现与部署**：AgentEscapeBench 包含 270 个实例，分为五个难度等级，支持完全自动化的评估。每个任务都涉及调用真实的外部函数、跟踪隐藏状态、传播中间结果并提交可确定性验证的最终答案。
- **评估结果**：实验对比了 16 种 LLM 代理和人类参与者。结果显示，随着依赖深度的增加，性能急剧下降。人类在难度为 5 时的成功率为 98.3%，在难度为 25 时降至 80.0%；而表现最好的模型从 90.0% 降至 60.0%。
- **与工作对比及分析**：轨迹分析表明，模型的失败主要归因于长程状态跟踪、线索遵循和中间结果传播的崩溃。这表明当前代理虽然能处理局部工具使用，但在处理深层上下文依赖方面仍存在显著困难，这与现有基准测试中观察到的“虚假”高性能形成对比。

#### 总结
AgentEscapeBench 填补了评估 LLM 代理在陌生环境中进行长程、工具 grounding 推理能力的空白。通过引入基于 DAG 的密室逃脱任务，该基准测试揭示了当前代理在状态跟踪和复杂依赖处理上的局限性，为未来提升代理的通用推理、行动和适应能力提供了重要的诊断测试床。
