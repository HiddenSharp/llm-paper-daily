## 文章做了什么
- **背景**：随着 GitHub Actions 和 n8n 等自动化平台越来越多地集成大型语言模型（LLM）代理以执行代码审查和数据同步等任务，形成了所谓的“代理工作流”（Agentic Workflows）。这种集成虽然提高了开发效率，但也引入了新的安全风险：攻击者可以通过构造特定输入（如 GitHub Issue 评论）来操纵 LLM 代理，导致凭证泄露或任意命令执行。
- **已有工作的不足**：现有的工作流分析方法主要依赖静态且对路径不敏感的分析来检测经典注入漏洞，无法推理可行的代理调用路径或运行时行为。另一方面， prior jailbreaking 研究通常假设 LLM 的输入是完全可控的，而在代理工作流场景中，攻击者只能控制基于工作流模板的部分提示词，且利用能力受到代理运行时能力和限制的约束，因此现有方法无法有效解决此问题。

## 文章的核心贡献点
- **提出了一个名为 JAW 的检测与利用框架**：这是首个针对托管在自动化平台上的代理工作流进行劫持的检测与利用框架，采用了一种称为“上下文 grounded 进化”（Context-Grounded Evolution）的新方法。
- **挑战1：如何确定触发代理调用的可行路径及输入约束？**
  - **难点**：工作流逻辑复杂，静态分析难以准确判断哪些路径在实际运行中是可达的，以及需要满足何种输入条件才能触发特定的代理行为。
  - **解决方法**：JAW 通过静态路径可行性分析（static path-feasibility analysis），识别出可行的代理调用路径以及触发这些路径所需的输入约束。
- **挑战2：如何在受限的输入控制下生成有效的攻击载荷？**
  - **难点**：攻击者无法完全控制 Prompt，且代理的执行能力受限于运行时环境，传统的越狱方法因假设输入完全可控而失效。
  - **解决方法**：JAW 结合动态提示来源分析（dynamic prompt-provenance analysis）和能力分析（capability analysis），确定输入如何被转换并嵌入 LLM 上下文，以及代理在运行时的可用动作和限制。在此基础上，JAW 迭代合成并优化（即进化）基于这些上下文的载荷，实现端到端的利用。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **评估结果**：在 GitHub 工作流和 n8n 模板上对 JAW 进行了评估。结果显示，4,174 个 GitHub 工作流和 8 个 n8n 模板可以被成功劫持，例如用于泄露用户凭证。
- **覆盖范围**：研究发现涵盖了 15 个广泛使用的 GitHub Actions，包括 Claude Code、Gemini CLI、Qwen CLI 和 Cursor CLI 的官方 GitHub Actions，以及两个官方 n8n 节点。
- **对比与影响**：作为该领域的首个研究，JAW 填补了代理工作流安全研究的空白。所有发现均已负责任地披露给受影响厂商，并获得了 GitHub、Google 和 Anthropic 等公司的确认、修复及漏洞赏金。

#### 总结
本文首次系统性地研究了自动化平台中代理工作流的安全风险，指出了现有静态分析和传统 LLM 越狱方法在此场景下的局限性。通过提出 JAW 框架，结合静态路径分析、动态提示追踪和运行时能力分析，实现了对代理工作流的有效劫持检测与利用。大规模实验验证了该风险的普遍性和严重性，并推动了主流厂商的安全修复。
