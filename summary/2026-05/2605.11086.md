## 文章做了什么
- **背景**：随着大型语言模型（LLM）和 AI Agent 能力的快速提升，其在网络安全领域的应用潜力巨大，但也带来了严峻的安全风险。其中，“漏洞利用”（Exploitation）是将一个已知的漏洞转化为具体安全影响（如未授权文件访问或代码执行）的关键能力。
- **已有工作为什么解决不了**：现有的网络安全基准测试主要侧重于漏洞复现、补丁生成或 CTF 解题，这些任务通常只需要源代码层面的推理。然而，真正的漏洞利用需要低级别的程序运行时推理（如内存布局、寄存器状态）、长期 horizon 的持续进展以及绕过多种现代防御机制（如 ASLR、栈保护等）。由于缺乏针对这一高阶能力的系统化评估，当前对 AI Agent 在真实攻击场景下的能力边界认识不足。

## 文章的核心贡献点
- **提出了一个 ExploitGym 基准测试框架**：这是一个大规模、多样化且真实的 AI Agent 漏洞利用能力评估基准，包含来自用户空间程序、Google V8 JavaScript 引擎和 Linux 内核的 898 个真实世界漏洞实例。
- **挑战1：低级程序推理与运行时适应的复杂性**。难点在于 Agent 需要理解堆元数据、栈帧和虚拟内存映射等底层细节，并在运行时动态调整策略。ExploitGym 通过提供可复现的容器化环境，要求 Agent 从触发漏洞的初始输入开始，逐步构建出完整的利用链，从而强制模型进行细粒度的底层推理。
- **挑战2：长程任务规划与现代防御机制的绕过**。难点在于利用过程往往需要串联多个原语（primitives）并绕过 ASLR、沙箱等缓解措施，这对 Agent 的长期记忆和逻辑连贯性提出极高要求。该基准通过在不同实例中变量化安全保护配置，隔离并量化了这些防御措施对 Agent 性能的具体影响，评估其在对抗环境下的真实成功率。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **实现与部署**：ExploitGym 包含 898 个实例，覆盖三个主要领域（用户空间程序、V8 引擎、Linux 内核）。所有配置均打包在可复现的容器化环境中，支持变量化安全保护设置。实验在禁用 safeguards 的可信访问程序下进行，以测量前沿模型的能力边界。
- **Evaluation 结果**：评估显示，尽管漏洞利用极具挑战性，但前沿模型仍能成功利用相当一部分漏洞。表现最强的配置是 Anthropic 的 Claude Mythos Preview 和 OpenAI 的 GPT-5.5，分别成功构建了 157 和 120 个实例的有效利用代码。
- **对比与发现**：与以往仅关注源码级任务的基准不同，ExploitGym 揭示了即使在启用广泛使用的防御机制的情况下，模型仍保持非 trivial 的成功率。这表明现有防御并非绝对安全，且 AI Agent 的进攻能力正在迅速逼近甚至突破某些传统防御假设。

#### 总结
ExploitGym 填补了 AI 安全评估中关于“漏洞利用”这一关键环节的空白。通过大规模真实场景的测试，论文证明了前沿 AI 模型已具备将漏洞转化为实际攻击的显著能力，即便在面对现代防御机制时也不例外。这不仅为防御性工作流提供了有价值的诊断工具，也突显了日益强大的 AI Agent 所带来的潜在网络安全风险，呼吁社区重视对此类双用技术的监管与防御研究。
