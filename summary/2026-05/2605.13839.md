## 文章做了什么
- **背景**：多智能体大语言模型（LLM）系统通常通过交换自然语言消息进行协作。这种接口虽然简单且可解释，但迫使发送方的中间计算结果被序列化为 token，接收方再重新处理这些 token。
- **已有工作的局限性**：这种基于文本的通信方式导致了巨大的生成 token 成本、预填充（prefill）开销以及 KV-cache 内存占用。随着智能体数量或通信轮次的增加，这种重复的“写入-读取”循环成为协作的主要成本瓶颈，且效率低下，因为模型间的通信被迫经过为人类设计的通道。

## 文章的核心贡献点
- **提出了一个 TFlow (Thought Flow)**：一种权重空间通信框架，用于已知且固定架构的接收方。它将发送方的隐藏状态编译为瞬态的、针对接收方的权重扰动，而非文本消息。
- **挑战1：如何避免将内部状态序列化导致的高昂预填充和 KV-cache 成本？**
  - **难点**：传统方法需将丰富的内部状态解码为 token，接收方再编码，导致上下文长度膨胀和计算冗余。
  - **解决方法**：TFlow 使用学习到的参数生成器，将发送方代理的内部激活映射为目标接收方模块的低秩 LoRA 扰动。这些扰动在接收方生成期间融合应用，无需扩大文本上下文，从而消除了额外的预填充步骤。
- **挑战2：如何实现实例级适应而不永久改变模型参数？**
  - **难点**：直接更新权重会破坏模型的通用性，且频繁微调成本高；而仅靠提示词无法充分利用发送方的深层语义信息。
  - **解决方法**：生成的 LoRA 扰动是瞬态的，仅针对当前查询在推理时应用，不永久修改模型权重。这实现了高效的实例级适应，同时保持了基础模型的完整性。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **实现细节**：使用三个 Qwen3-4B 代理作为实验对象。发送方代理经过角色提示冻结处理，参数生成器将激活转换为低秩扰动。
- **评估结果**：
  - 在五个基准测试中，TFlow 相比独立的接收方代理，准确率最高提升 8.5 个点，同时减少高达 32.69% 的处理 token 数。
  - **与相关工作对比**：与基于文本的三智能体基线相比，TFlow 将总处理 token 数减少了高达 83.27%，墙钟推理时间减少了高达 4.6 倍。
  - 在五个基准中的四个上保持了具有竞争力的准确率，证明了瞬态低秩权重扰动作为高效多智能体协作介质的有效性。

#### 总结
本文指出了当前多智能体 LLM 系统中基于自然语言通信的效率瓶颈，提出了 TFlow 框架，通过将通信内容转化为权重空间的低秩扰动（LoRA），显著降低了计算和内存开销。实验表明，该方法在保持甚至提升准确率的同时，大幅减少了 token 消耗和推理延迟，为高效的多智能体协作提供了新的范式。
