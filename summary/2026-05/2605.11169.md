## 文章做了什么
- **背景**：大型语言模型（LLM）代理在部署环境中通常以 ReAct 风格运行，交替进行推理、动作选择和观察以解决多步决策任务。在重复处理相关任务的场景中，微小的动作选择错误会累积，导致工具调用浪费、延迟增加和可靠性降低。
- **已有工作为什么解决不了**：现有的推理时适应方法主要依赖提示工程（Prompting）或检索增强，这些方法通过上下文操作间接影响行为。它们缺乏一个显式的决策层来对候选动作进行评分、表示不确定性或根据动作级别的反馈进行在线更新，因此无法提供可追踪、细粒度且具备不确定性意识的部署时适应支持。

## 文章的核心贡献点
- **提出了一个 OLIVIA 框架**：一种用于 ReAct 风格代理的推理时动作适应框架，将 LLM 的最终动作选择层建模为基于候选动作的上下文线性 Bandit 算法。
- **挑战1：如何在保持底层推理过程不变的情况下，实现直接且高效的动作策略适应？**
  - **难点**：传统微调成本高且破坏通用性，而提示调整间接且难以量化优化。
  - **解决方法**：OLIVIA 利用冻结的隐藏状态作为决策上下文，直接在动作选择接口处引入轻量级的线性 Bandit 层，仅针对动作选择进行在线更新，从而保留原始推理能力并实现低开销适应。
- **挑战2：如何在缺乏大量标注数据的情况下，有效平衡探索与利用以提升样本效率？**
  - **难点**：部署环境中的反馈稀疏且带有噪声，盲目利用已知最佳动作可能导致局部最优，而过度探索会增加错误成本。
  - **解决方法**：引入上置信界（Upper-Confidence-Bound, UCB）探索策略，结合显式的不确定性估计，使代理能够根据动作级别的反馈高效地改进策略，最小化计算开销的同时提升决策质量。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **实现与部署**：OLIVIA 被实例化并在四个基准测试上进行评估。该框架无需更新 LLM 参数，仅维护一个轻量级的线性模型用于动作评分和选择，适合流式部署。
- **评估结果**：实验显示，OLIVIA 在所有基准测试中均一致地优于静态 ReAct 基线和基于提示的推理时适应基线。
- **对比优势**：与依赖上下文操纵的提示或检索方法相比，OLIVIA 提供了显式的在线决策层，能够更有效地利用动作级反馈，显著减少了因早期错误导致的无效轨迹，提高了任务完成率和效率。

#### 总结
OLIVIA 证明了在 LLM 代理中引入显式的在线决策层是纯提示或检索基于适应方法的有效替代方案。通过将动作选择建模为上下文线性 Bandit 问题，OLIVIA 实现了细粒度、可追踪且具备不确定性意识的推理时适应，为部署环境下的 LLM 代理提供了高效、低开销的性能提升路径。
