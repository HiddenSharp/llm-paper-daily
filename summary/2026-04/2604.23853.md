## 文章做了什么
- **背景**：LLM Agent 可以通过从执行轨迹中蒸馏出可重用的“技能”（结构化指令包）来提升性能，而无需修改模型权重。然而，现有的技能蒸馏流程缺乏一个关键信号：每一步骤的具体成本。
- **已有工作为什么解决不了**：现有的蒸馏方法通常将轨迹简单划分为成功和失败两类并提取规则，无法区分“添加缺失步骤以修复错误”与“移除昂贵但无关的步骤以降低成本”。虽然 LangSmith、Langfuse 等现有可观测性工具能追踪 token 数量和成本，但它们主要面向人类操作员的仪表盘分析，未能提供紧凑的、可供蒸馏管道直接消费的中间表示形式（如包含每步成本、冗余标记的结构化数据）。

## 文章的核心贡献点
- **提出了一个 ClawTrace 平台及 CostCraft 蒸馏管道**：ClawTrace 是一个开源的 Agent 追踪平台，能将会话编译为紧凑的 YAML 格式 TraceCard；CostCraft 基于此生成三种技能补丁（保留、剪枝、修复）。
- **挑战1：缺乏细粒度的成本归因导致无法优化效率**。难点在于难以识别哪些高成本步骤对结果无实质贡献。论文通过 ClawTrace 记录每个 LLM 调用、工具使用和子 Agent 生成的具体 USD 成本和 token 计数，并在 TraceCard 中标记冗余，使管道能精准定位浪费。
- **挑战2：蒸馏出的技能在跨任务泛化时存在不对称性**。难点在于特定于基准测试的“保留规则”可能在新任务上导致性能倒退。论文通过引入“剪枝补丁”（Prune patches），利用反事实论证移除无效的高成本步骤，发现剪枝规则具有更好的跨基准迁移能力，而保留规则则需谨慎使用。

## 实现与部署，evaluation 结果，要有和相关工作对比
- **实现与部署**：ClawTrace 通过八个事件钩子检测 Agent 会话，支持将子 Agent 会话链接回父跨度，并按实际计费率核算缓存 token。它编译生成约 1.5 kB 的 TraceCard，包含每步美元成本、各类 token 计数、冗余工具调用簇及失败/修复标记。API 接受纯 JSON，无框架依赖。
- **评估结果**：
    - 在 30 个留存的 SpreadsheetBench 任务上进行消融实验，显示成本归因和剪枝补丁均能独立减少质量回归。
    - 在 30 个不相关的 SkillsBench 任务上测试时发现不对称现象：剪枝规则跨基准迁移并将中位数成本降低了 32%；而针对特定基准约定训练的保留规则在新任务类型上导致了性能回归。
- **对比**：相较于仅提供人类可读仪表盘的现有工具（LangSmith 等），ClawTrace 提供了机器可读的紧凑中间表示，专门服务于自动化技能蒸馏。

#### 总结
本文指出了当前 LLM Agent 技能蒸馏中忽视成本信号的缺陷，提出了 ClawTrace 追踪平台和 TraceCard 数据格式，解决了细粒度成本归因难题。基于此构建的 CostCraft 管道生成了保留、剪枝和修复三类技能补丁。研究揭示了剪枝规则在跨任务成本优化上的高效性与泛化能力（降低 32% 中位数成本），而保留规则可能存在过拟合风险。ClawTrace 作为开放基础设施发布，旨在推动成本敏感的 Agent 研究。
